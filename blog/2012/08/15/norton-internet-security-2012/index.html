Norton Internet Security & Norton AntiVirus 2012 19.8.0.14 Final [Update]

Aug 15th, 2012

14 августа 2012 года - Norton™ Internet Security 2012 & Norton™ AntiVirus 2012 обновились до 19.8.0.14. Обновление происходит автоматически через Live Update. Обновлены дистрибутивы до версии 19.8.0.14 (ссылки под катом).

Версия 19.8.0.14 имеет улучшения и исправления, вот часть из них:

<code>
• Добавлена совместимость с Windows 8.
• Повышена производительность для работы с Internet Explorer 10.
• Исправлена ошибка в Symantec Service Framework, которая обнаруживалась при изменении правил файрвола (<a href="http://community.norton.com/t5/Norton-Internet-Security-Norton/NIS2012-crashing-with-changes-in-firewall/td-p/589474" target="_blank">forum thread</a>)
• Исправлена ошибка сканирования внешних носителей большой емкости, антивирус при этом отображал следущее: "просканирован 1 файл".
• Исправлена ошибка в работе модуля "Identity Safe Cards".
 
Этот патч включает в себя также исправления предыдущих версий патчей. Вот часть из них:
 
• Corrected the Google Chrome-specific issue where Norton Toolbar does not load when Google Instant is ON (<a href="http://community.norton.com/t5/Norton-360/Bug-in-Chrome-Toolbar/m-p/665133/highlight/true#M66999" target="_blank">link</a>)
• Corrected the issue where customer is logged out of IDS despite setting a log out time of 15 mins (<a href="http://community.norton.com/t5/Norton-Identity-Safe-Public-Beta/Identity-Safe-Times-Out/m-p/663673#M164" target="_blank">link</a>)
• Better handling of login sites where username / password span multiple inputs
• Corrected an issue where the Password field was getting saved with "Password" string instead of customer input
• Corrected the issue where All logins listed in toolbar didn’t get refreshed unless the open browser session was restarted
• Corrected the issue where it was unable to change Identity Safe password
• Long passwords are accepted in recent Online Vault UI (<a href="http://community.norton.com/t5/Norton-Internet-Security-Norton/Identity-Safe-issues-with-NIS-2012/m-p/538844" target="_blank">link</a>)
• Corrected the issue where customers remain logged into Norton Account even though they had unchecked Remember Password option from Create New Norton Account UI
• Corrected the issue where customers were not prompted to fill the card details using the saved data in IDsafe in My Account page (<a href="http://community.norton.com/t5/Norton-Internet-Security-Norton/Lost-online-identity-vault/m-p/641493/highlight/true#M191420" target="_blank">link</a>)
• Corrected the issue where customer’s login got filled in the Edit Phone Numbers page in capitalone.com
• Corrected the issue where Last Submitted Login got autofilled when trying to overwrite the filled login (<a href="http://community.norton.com/t5/Norton-Internet-Security-Norton/Identity-Safe-autofill-problem/m-p/695413/highlight/true#M200655" target="_blank">link</a>)
• Corrected the issue where Update Password infobar does not appear when password is attempted to change for a saved login</code>
